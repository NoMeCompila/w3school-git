Git y Github

¿Qué es Git?

Git es una tecnología usa da para el control de versiones en el desarrollo de software, fue creado por Linus Torvalds y está pensado para el trabajo remoto en equipo y tener control total sobre las distintas versiones de un producto. Puede usarse de manera local, pero su gran poder radica en manejar repositorios remotos con ayuda de plataformas como por ejemplo GitHub o GitLab.

Principales Comandos

Comando Descripcion
git --version Muetra en pantalla la versión de git que tenemos instalado
git init Inicia un repositorio local de git
git config user.name "nombre de usuario" configura el nombre del usuario del repositorio local
git config user.email "email de usuario" configura el email del usuario del repositorio local
git status mustra el estado de los archivos y directorios del proyecto, en verde los trackeados y rojos los untracked
git add "archivo" añade un archivo al stagin area se puede modificar con -A para agregar todos de una sola vez
git commit -m "cometario" Confirma los cambios y genera un "punto de guardado" o commit area
git log Para ver el historial de commits en el repositorio local
git help --all Muestra la documentacion para usar los comandos de git
git branch "nueva-rama" Crea una nueva rama de trabajo
git branch Lista todas las ramas existentte es un repositorio local
git checkout "nueva-rama" Sirve para moverse entre ramas
git merge "nueva rama" Une 2 ramas existentes en una sola generalmente la master
git branch -d "nueva-rama" Sirve para borrar una rama
git remote add origin "url" Agrega un repositorio remoto
git push --set-upstream origin master sube los cambios al repositorio remoto
git pull origin Actualiza el repositorio local
git push origin Sube los cambios al repositorio remoto
git clone URL Clona un repositorio remoto al local
git revert HEAD Mueve el puntero HEAD al commit anterior
git reset "hexa-key" Mueve el puntero HEAD al commit que se indica para revertir cambios
